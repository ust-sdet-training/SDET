During this project, I developed a complete API Automation Framework for a Retail Application using Java, Rest Assured, JUnit 5, Maven, MySQL, and JDBC. The main goal was to create a scalable and maintainable automation framework that could efficiently test backend services and validate data across different layers of the application.

To achieve this, I designed reusable framework components such as authentication utilities, request and response specifications, configuration classes, and base test classes. These components helped reduce code duplication and made the framework easier to maintain and extend. Configuration details such as API URLs, user credentials, OAuth credentials, API keys, and database settings were managed through environment variables and configuration files instead of being hardcoded, making the framework more flexible across different environments.

I automated test scenarios for key modules of the retail application, including Login, OAuth Authentication, Products, Cart, Orders, Secure Orders, Partner APIs, and Legacy XML APIs. The test suite covers both positive and negative scenarios, including successful logins, invalid credentials, unauthorized access, role-based access control, product search and filtering, cart operations, order creation, and complete order lifecycle workflows. Different HTTP status codes such as 200, 201, 204, 400, 401, 403, 404, 409, and 423 were validated to ensure the APIs responded correctly under various conditions.

To improve test reliability and ensure contract validation, I implemented JSON Schema Validation for login, OAuth, product, cart, order, secure order, and error responses. This ensured that API responses always matched the expected structure. For legacy XML services, I implemented XML Schema (XSD) validation to verify that XML responses complied with the defined schema.

In addition to API validation, I integrated database validation using JDBC. After performing API operations such as creating, allocating, and shipping orders, I directly queried the MySQL database to verify that the data was stored correctly and that status changes were accurately reflected in the backend. This enabled complete end-to-end validation by comparing API responses with actual database records.

Overall, this project provided hands-on experience in building a real-world backend automation framework that combines API testing, authentication testing, schema validation, XML validation, and database verification. The framework follows industry-standard automation practices and demonstrates strong principles of reusability, maintainability, scalability, and end-to-end quality assurance.
