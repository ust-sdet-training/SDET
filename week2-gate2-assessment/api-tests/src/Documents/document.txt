During this project, I built a complete API Automation Framework for a Retail Application using Java, Rest Assured, JUnit 5, Maven, MySQL, and JDBC. I created reusable components such as authentication utilities, request and response specifications, configuration classes, and base test classes to make the framework easy to maintain and scale. Instead of hardcoding values, I used environment variables and configuration files to manage URLs, user credentials, OAuth credentials, API keys, and database connections.
I automated testing for major application features including Login, OAuth Authentication, Products, Cart, Orders, Secure Orders, Partner APIs, and Legacy XML APIs. The test suite covers both positive and negative scenarios such as successful logins, invalid credentials, unauthorized access, role-based access control, product searches, cart operations, order creation, and complete order lifecycle flows. I also validated different HTTP response codes like 200, 201, 400, 401, 403, 404, 409, and 423 to ensure the APIs behave correctly under different conditions.
To improve reliability, I implemented JSON Schema Validation for login, OAuth, product, cart, order, secure order, and error responses to verify that API responses always follow the expected structure. For legacy XML services, I performed XML Schema (XSD) validation to ensure the XML responses were correctly formatted.
I also integrated database validation using JDBC. After performing API operations such as creating and updating orders, I directly queried the MySQL database to verify that the data was stored correctly and that order status changes were accurately reflected in the backend. This allowed me to validate both the API layer and the database layer as part of a complete end-to-end testing approach.
Overall, this project helped me build a real-world backend automation framework that combines API testing, authentication testing, schema validation, XML validation, and database verification while following industry-standard practices for reusability, maintainability, and scalability.
