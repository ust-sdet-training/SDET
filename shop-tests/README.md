# Playwright Test Suite

This project contains automated test cases written using **Playwright** for validating the functionality and accessibility of the application.

## Test Files

### 1. `accessibilityCheck.spec.ts`

This file contains **2 test cases** related to accessibility testing using **AxeBuilder**.

#### Test Case 1: Full Accessibility Scan
Performs a complete accessibility scan on the following pages:

- `/` (Home Page)
- `/catalog`
- `/a11y-lab`

The test verifies that the pages comply with accessibility standards by running a full Axe accessibility audit.

#### Test Case 2: Critical and Serious Accessibility Issues
Performs an accessibility scan on the **Home Page (`/`)** and validates that there are **no Critical or Serious accessibility violations**.

---

### 2. `example.spec.ts`

This file contains the default example test code generated by **Playwright** when the project is initialized using:

```bash
npm init playwright@latest
```

It serves as a reference for basic Playwright test structure and syntax.

---

### 3. `happy-path.spec.ts`

This file contains the **end-to-end happy path scenario** of the application.

The flow includes:

- User Login
- Product Selection
- Add Product to Cart
- Checkout Process
- Order Placement
- Order Validation

This test verifies that the primary user journey works successfully without any errors.

---

### 4. `login.spec.ts`

This file contains test cases for validating different login scenarios.

#### Test Cases

- ✅ Valid Login
- ❌ Invalid Login
- 🔒 Locked User Login

These tests ensure that the authentication functionality behaves as expected for different user types and credentials.

---

### 5. `search-to-cart.spec.ts`

This file contains test cases related to product search and cart functionality.

#### Test Cases

- Search for an existing product
- View an existing product
- Search for a product and add it to the cart

These tests validate the product discovery and shopping cart workflow.
